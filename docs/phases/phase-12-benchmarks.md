# Phase 12 — Benchmarks

**Goal:** automated benchmark harness with reference workloads, criterion micro-benches, and public perf tracking.

**Estimated effort:** 3–4 days.

**Prerequisites:** Phase 11 complete (so end-to-end works).

**Reading:** [`docs/research/benchmarks.md`](../research/benchmarks.md).

**Output:** `cargo bench` produces HTML report. Nightly CI publishes results to gh-pages. Reference workload hits targets.

## Checklist

### Benchmark crate

- [ ] `benchmarks/Cargo.toml` — new workspace member.
- [ ] `benchmarks/benches/` — criterion harnesses.
  - [ ] `parse.rs` — formula parsing throughput.
  - [ ] `arithmetic.rs` — `+`/`*`/`/` per-op throughput.
  - [ ] `lookup.rs` — hash lookup throughput on 10k-row table.
  - [ ] `aggregate.rs` — SUM/COUNT over 400k-cell column.
  - [ ] `end_to_end.rs` — full pipeline at each tier.
  - [ ] `quick.rs` — smoke version that runs in CI per-PR (< 30 s).
- [ ] `benchmarks/fixtures/` — committed small fixtures, generator scripts.

### Fixture generation

- [ ] `benchmarks/src/bin/generate-fixtures.rs` — deterministic (seeded) xlsx generation for all tiers.
- [ ] Fixtures not committed (too large); regenerated by CI before benches run.
- [ ] Golden outputs for correctness validation.

### Reference-workload bench

- [ ] `end_to_end.rs` includes the reference 400k-row benchmark.
- [ ] Reports RSS + wall-clock.
- [ ] Target asserted in-test (not just reported): fails if > 250 MB or > 3 min.

### CI integration

- [ ] `.github/workflows/ci.yml` runs `cargo bench --bench quick` on every PR.
- [ ] `.github/workflows/nightly.yml` runs full bench suite; publishes with `benchmark-action/github-action-benchmark@v1`.
- [ ] PR regression threshold: 10% RSS, 20% wall-clock.

### Comparative numbers

- [ ] Document comparative results in `docs/research/benchmarks.md`.
- [ ] Update with real measured numbers once Phase 10+ stabilises.

### Memory measurement

- [ ] Integrate `memory-stats` crate for cross-platform peak RSS.
- [ ] Tests assert peak RSS within budget.

## Done when

- Reference workload passes its assertions.
- Comparative numbers vs formualizer documented.
- Nightly CI publishes benchmark results to gh-pages.
- Per-PR smoke benchmarks run in < 30 s.
